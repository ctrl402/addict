实时优化（RTO）是过程工业"闭环利润最大化"的核心环节：它位于生产计划调度与先进过程控制（APC/MPC）之间， 以严格的稳态机理模型为内核，把"当前工况下经济最优的操作点"实时计算出来并下发给控制器， 使装置在原料、产品牌号、环境与设备状态不断变化的情况下，持续逼近经济最优。 本文基于本室 RTO 专题库（含 Biegler、Shao、Bonvin/Marchetti、Engell、Naysmith、Yip、Papasavvas 等学派的一手学位论文与专著）撰写。

一、背景：目的意义、技术框架与流程要点

1.1 目的与意义

过程工业（炼油、石化、化工、化肥、电力、冶金）的利润对操作点高度敏感。一套千万吨级炼油装置， 操作点偏离最优 1%，一年就是千万级人民币的损失。传统 APC/MPC 只能把过程"稳"在给定设定值上， 而设定值本身是否经济最优，需要由 RTO 来回答。RTO 的本质是： 在模型—现实存在失配、工况不断扰动的前提下，周期性地求解一个以经济效益为目标的稳态优化问题， 并把求得的最优操作目标（温度、流量、压力、塔顶/塔底采出等）作为 APC 的新设定值。

行业经验表明，成功实施的 RTO 通常带来 能耗下降 1%–3%、毛利提升 2%–5%， 投资回收期多在 1 年以内（ARC Advisory Group 多份报告口径），是过程工业"存量挖潜"性价比最高的数字化手段之一。

1.2 技术框架：分层优化控制体系

RTO 不是孤立模块，而是分层优化控制金字塔的中间层：

层级时间尺度任务典型工具
计划 / 调度天–周资源分配、排产、原料与产品决策LP / MILP 计划优化
RTO（实时优化）小时–班次基于严格模型的稳态经济优化，求最优操作点联立方程稳态优化器（RSQP 类）
APC / MPC分钟–秒把 RTO 目标稳在约束边界内，抗扰动多变量预测控制
基础回路控制秒流量/温度/压力单回路调节PID

关键信息流是自上而下的目标传递与自下而上的数据汇聚： 现场仪表 → 实时数据库 → RTO（数据整定 + 参数估计 + 优化求解）→ 设定值 → APC → 回路。

1.3 流程要点：RTO 的迭代闭环

一个标准 RTO 迭代（周期通常 1–8 小时，依稳态检测而定）由以下环节串联：

① 稳态检测（Steady-State Detection）：判断过程是否已进入稳态（滑动窗口方差检验、零阶/一阶假设检验），只有稳态数据才用于优化，避免动态噪声污染。
② 数据整定 / 数据协调（Data Reconciliation）：用物料/能量/动量守恒约束，对带噪声、甚至矛盾的测量值做加权最小二乘估计，消除冗余测量冲突、检出粗大误差（gross error）。
③ 参数估计 / 模型更新（Parameter Estimation / Model Update）：用协调后的数据反估模型参数（如塔效率、传热系数、动力学常数），使严格模型"贴合"当前装置。
④ 稳态经济优化（Steady-State Optimization）：以更新后的严格机理模型为等式约束、以经济效益（利润/能耗）为目标，求最优操作点——这是 RSQP 等大规模 NLP 求解器大显身手之处。
⑤ 目标计算与下发：将优化变量转化为 APC 可读的设定值，经可行性/合理性校验后下发；同时用"信任窗口（trusted window）"限制单次移动幅度，保障平稳过渡。

1.4 两条技术路线

  • 模型驱动 RTO（Model-based RTO）：依赖完整参数估计使模型贴合现实，再解优化。优点是可解释、能给出最优结构；难点在参数估计收敛性与模型失配。
  • 修正自适应 RTO（Modifier Adaptation, MA）：不直接追求参数精确，而是在 KKT 条件上叠加"修正项"（梯度/函数值修正），用实测输入输出直接补偿模型—现实差距，无需完整参数估计、对稳态检测要求更低。代表：EPFL Bonvin 组、Marchetti 2009、Oliveira da Silva 2023。
  • 免稳态检测 RTO（ROPA 等）：以"持续参数自适应"替代显式稳态检测，适合频繁扰动工况。代表：USP Matias 2018、Carneiro 2020。

2.4 国内实施情况（2023–2025 最新）

起步与历史：中国 RTO 起步于 1990 年代，中石化率先试点，最早部署由 Setpoint（1996 并入 AspenTech）完成；此后 Honeywell、AspenTech 在国内炼化装置实施了数百个项目。但据 ARC 早年观察，到 2005 年前后国内"真正良好运行"的 RTO 仅约三分之一，瓶颈多在模型维护、数据质量与参数更新闭环。

市场格局（IDC 2023）：据 IDC《中国石油化工智能工厂解决方案市场份额, 2023》，中国石化智能工厂解决方案市场规模超 36 亿元，RTO/在线优化是其中的高价值软件层。份额前七：石化盈科（23.9%，近 1/4，居首）、中控技术、昆仑数智为前三，和利时、施耐德、西门子、霍尼韦尔列四至七（中化新网/IDC）。

政策驱动：工信部 2024 年《工业重点行业领域设备更新和技术改造指南》提出，到 2027 年数字化转型成熟度 3 级及以上企业占比达 15%+、4 级及以上达 7%+、关键工序数控化率 85%+。而目前成熟度 3 级及以上的数字化炼厂不足 10%、4 级寥寥，且大量国有大中型炼厂设备运行超 30 年——RTO/APC 的国产化改造空间巨大。

2.5 实施效果与常见落差

成功项目普遍回报丰厚（能耗 −1%~−3%、毛利 +2%~+5%、回收期 <1 年）； 但失败/闲置项目同样普遍，根因通常不是优化算法，而是： ① 实时数据质量差、协调与稳态检测失效；② 模型长期不更新（"建完即弃"）； ③ RTO 与 APC 目标脱节；④ 缺少持续运维机制。这正是学术界"修正自适应 / 免稳态检测 / 自主运行"等方向要解决的问题。

三、国内外学术界的创新贡献 附参考文献

3.1 大规模优化求解器：从 Reduced Hessian 到 RSQP

RTO 的内核是一个带大规模代数/微分方程约束的 NLP。1980–2000 年代， Lorenz Biegler（CMU）系统建立了 reduced-Hessian / reduced-space SQP（RSQP）理论， 区分"优化变量"与"模型等式约束"，用零空间法 + BFGS 近似降维 Hessian，使千/万变量级流程优化得以在单机求解 （AIChE J 1982 不可行路径 SQP、CCE 1983–1999 一系列 reduced-Hessian / multiplier-free / interior-point 工作，本室均存档）。

邵之江（浙大）将降维思想推向"简约空间 SQP（Reduced-Space SQP）"并实现工程化， 面向在线优化提出计算策略（Shao 等, 1997/1998/1999），成为国内工业 RTO 求解器的算法底座。

3.2 稳态检测与数据整定

Naysmith（Waterloo, 1998）系统比较了 RTO 中"闭式方程"与"开式方程"两类表述对优化—控制一体化求解的影响，是奠基性文献。 万百五、黄正良《大工业过程计算机在线稳态优化控制》是国内该方向的经典专著，系统论述稳态优化控制的原理与实现。 数据协调（Data Reconciliation）作为 RTO 前置环节，与参数估计耦合，是鲁棒性的第一道关口（本室亦藏清华李博 2001 博士论文专攻数据校正）。

3.3 模型更新与参数估计

Wai San Yip（McMaster, 2002，导师 T. E. Marlin）的博士论文《Model Updating in Real-Time Optimization》 专门研究 RTO 中的模型更新策略，并探讨 RTO 与过程控制的集成。 USP 团队（Palacio 2013 数据分析、Menezes 2016 丙烯 VRD 装置 Petrobras 应用）则把参数估计与工业数据实践紧密结合。

3.4 修正自适应（Modifier Adaptation, MA）

Bonvin 团队（EPFL）提出"自优化控制 + 修正自适应"框架：用实测梯度/函数值修正模型 KKT 条件，使最优解收敛到真实最优， 而无需精确参数估计。Marchetti（EPFL, 2009）的博士论文《Modifier-adaptation methodology for real-time optimization》是该方向的里程碑。 后续 Oliveira da Silva（Valladolid, 2023）把 MA 与经济 MPC 融合；Krishnamoorthy（NTNU/SINTEF, 2019）研究不确定性下的在线优化与反馈 RTO。

3.5 免稳态检测 RTO / ROPA 与 KPI 导向

Matias（USP, 2018）提出 ROPA（Persistent Parameter Adaptation），以持续参数自适应替代显式稳态检测； Carneiro（USP, 2020）进一步给出 ROPA 在线参数估计实现；Câmara（UFRJ, 2018）研究 RTO 的辨识技术； Santos（UFRGS, 2021）提出"KPI 导向 RTO（KPI-RTO）"，以业务指标而非纯模型驱动优化目标。

3.6 自主运行（Autopilot）与耦合过程

Papasavvas（Edinburgh, 2018）的《From Real-Time Optimization Techniques to an Autopilot for Steady-State Processes》 主张把 RTO 与控制融合为稳态过程的"自动驾驶"；Himmel（OvGU Magdeburg, 2021，Sundmacher/Engell/Sager 组） 研究耦合过程的基于优化运行策略；Serralunga（UNL, 2014）面向工业过程与能源系统的 RTO。

3.7 国内高校团队（补充）

  • 浙江大学（邵之江组）：降维 SQP、在线优化、记忆增强型 RTO（本室收藏江爱朋/王可心/方学毅/陈伟锋/张正江/李健舒/姚科田/羊城/游江洪/刘晨光等十余篇学位论文）。
  • 清华大学（陈丙珍组）：过程系统工程、数据校正（李博 2001 博士）。
  • 大连理工、中国石油大学（北京，罗雄麟组）：集成优化、不确定性下在线优化（杨斌博士等）。

四、技术深究：基于大规模联立方程稳态严格机理模型的 RTO

本节聚焦用户关注的技术内核——以大规模联立方程稳态严格机理模型为载体，融合高效鲁棒优化求解、数据整定与参数估计，以 RSQP 为主要求解器的 RTO 技术栈，以及工程落地中的求解策略与实施技巧。

4.1 联立方程稳态严格机理模型（Equation-Oriented, EO）

"严格机理模型"指由质量/能量/动量守恒、相平衡、反应动力学、设备关联等第一性原理方程构成的装置全模型。 与传统的序贯模块法（Sequential Modular, SM）——逐个单元依次计算、靠撕裂流迭代收敛——不同， RTO 内核普遍采用联立方程法（Equation-Oriented）：把全装置所有方程联立成一个大型非线性代数系统 F(x, y, p) = 0（x 优化/操作变量，y 模型状态/流股变量，p 参数），一次性求解。

EO 的优势对 RTO 至关重要：

  • 灵敏度可解析获得：通过联立方程的 Jacobian，可直接得到目标/约束对操作变量与参数的导数，供 SQP 使用；
  • 天然支持优化嵌套：模型等式即优化约束，无需"外面套一层迭代"；
  • 便于分解与并行：可整体求解，也可按单元/工段分解（-Schur 补/拉格朗日松弛）。

4.2 高效鲁棒优化求解：RSQP 为何是主力

RTO 的 NLP 形如：

min  Φ(x)  （经济效益，如利润/能耗）
s.t.  F(x, y, p) = 0  （联立稳态严格模型，大规模）
     g(x, y) ≤ 0  （操作/质量/设备约束）
     xL ≤ x ≤ xH

直接把全部变量（x ∪ y）当优化变量在满空间求解，Hessian 维度爆炸且病态。 RSQP（Reduced-Space SQP）的核心思想：

  • 变量分离：把 y（由模型方程隐式决定）从优化变量中"消掉"，只把 x 作为独立优化变量（reduced variables）；
  • 零空间法 + 降维 Hessian：在每次 SQP 迭代，用模型 Jacobian 的零空间把搜索方向投影到可行流形，并用 BFGS 仅在降维空间近似 Hessian（reduced Hessian），规模从"全变量数万"降到"自由度数百"；
  • 不可行路径（Infeasible Path）：允许中间迭代违反模型方程，靠罚函数/约束逐步收紧，比"每步先解算稳态"的 feasible-path 更快更鲁棒（Biegler 1982/1985；Shao 1999 用于在线优化）；
  • 内点 / multiplier-free 变体：对不等式约束多的炼化模型，内点法（Biegler 1999）与免乘子法（Biegler 1998）进一步改善数值稳定性。

这也是 AspenTech、AVEVA ROMeo 等商业 RTO 内核的算法底色——RSQP 不是某一个软件，而是一类被工业验证的大规模流程 NLP 求解范式。

4.3 数据整定 / 数据协调（Data Reconciliation）

鲁棒性的第一道关。以守恒方程 A·m = 0（m 测量向量，A 关联矩阵）为约束，做加权最小二乘：

min  (m − m̂)T W (m − m̂)
s.t.  A·m̂ = 0

要点：① 用测量方差定权 W；② 通过全局检验与节点检验检出粗大误差（传感器故障/误标定量程）； ③ 对非线性/双线性系统做迭代协调；④ 协调后的数据才是参数估计与优化的可信输入。

4.4 参数估计（Parameter Estimation / Model Update）

用协调后的稳态数据反估模型参数 p（塔效率、传热/传质系数、动力学常数、催化剂活性等）：

min  Σk ‖ h(xk, yk, p) − zk ‖²  s.t. F(xk, yk, p)=0

该问题常与数据协调耦合（同步协调 + 估计），并受可辨识性限制——不是所有参数都能被有限测量唯一确定， 这正是 Yip(2002) 等研究"模型更新策略"的动机：何时更新、更新哪些参数、如何避免过拟合噪声。

4.6 工业实施技巧（来自一线与专著的"know-how"）

数据侧：稳态检测用滑动窗口 + 假设检验；设最小稳态持续时长，过滤过渡工况；数据先做协调与粗大误差剔除，再做参数估计。
模型侧：用上一周期解做热启动（warm start），避免冷启动失败；对强非线性单元设合理初值；参数更新加"信任窗口"限制单步幅度，防止模型跳变。
优化侧：自由度选择宁少勿滥（多一个自由度多一份震荡风险）；约束尽量用软约束 + 罚项保鲁棒；对关键瓶颈用 LP 预规划结构、再用 NLP 精算。
闭环侧：RTO 目标经合理性校验再下发；与 APC 设定值"对齐"，避免 RTO 与 MPC 打架；建立"投用率/收敛率"KPI 做持续运维。
组织侧：RTO 是"建得成、更要用得久"——模型维护机制与数据质量治理，比算法本身更决定成败（呼应 2.5 节的落地落差）。
